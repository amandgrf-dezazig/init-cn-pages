# init-cn-pages

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库用于存放和托管一系列独立的 HTML 页面。这些页面可能涉及不同的主题或功能，彼此之间没有强关联。仓库的主要目的是提供一个集中存放和静态发布 HTML 内容的空间，方便归档、展示和分享。

## 目录说明

```
├── pages/               # 存放所有 HTML 页面的主目录
│   ├── example1/        # 示例页面子目录（按需创建）
│   │   └── index.html
│   ├── example2/
│   │   └── index.html
│   └── ...
├── assets/              # 公共资源文件（可选）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md            # 本文件
```

- `pages/`：所有独立的 HTML 页面均放置在此目录下，每个页面可拥有独立的子目录，以便组织其相关资源。
- `assets/`：存放多个页面可能共用的样式、脚本或图片等资源，避免重复。

## 页面归档说明

- 每个 HTML 页面应尽量保持自包含，所需资源优先放在其子目录内，或引用 `assets/` 下的公共文件。
- 页面文件名建议使用 `index.html`，便于直接通过目录路径访问。
- 仓库不限定页面的具体用途，但请确保内容合法合规。

## 维护说明

- 欢迎提交包含独立 HTML 页面的 Pull Request，新增页面请在 `pages/` 下创建新的子目录。
- 请确保新增页面不包含恶意代码、跟踪脚本或违反平台政策的内容。
- 本仓库主要用于归档和静态展示，不建议用于动态服务或频繁更新的场景。
- 如需删除或修改已有页面，请通过 Issue 或 Pull Request 提出。
